title: 入口点包装题复现r-png
date: 2026-2-7

这道题提示图片png格式，推测是根据所给的文件反推代码并最后得出图片
看main函数可知这段反编译代码展示了一个典型的入口点包装器（Entry Point Wrapper）模式，其中出现了三个变量
sub_16A40：这很可能是真正的逻辑入口（flag 检查逻辑通常在此）
sub_24760：检查是否为运行时初始化（Go runtime / C runtime）
qword_59E78：查看交叉引用，可能是关键的全局状态或配置
我先看了24760，经过分析发现这段代码的关键作用是设置程序运行环境，最终调用用户编写的真实 main 函数。，而其中的v32 = (*(int (__fastcall **)(__int64 (__fastcall **)()))(v41 + 40))(p_sub_16A40_1);
p_sub_16A40_1 = 用户编写的 main 函数（即之前提到的 sub_16A40），由此可以知道这关键在于16a40
这是典型的 Rust 编写的文件加密工具，核心逻辑是 RC4 流密码 的变种。
关键识别特征

  1. 算法：修改版 RC4
    密钥调度（KSA）：标准 RC4 初始化（256 次循环，初始化 S-box）
    伪随机生成（PRGA）：标准 RC4 流程，但每个生成的密钥流字节 +69（0x45） 后再与明文异或：
    c
    复制
    buf_1[v13] ^= (keystream_byte + 69);
  2. 密钥约束
    4 位纯数字（如 0123、9999）
    程序会显式检查：”[!] key 必须是 4 位数字，比如 0123\n”
  3. 文件操作流程
    输入：程序从第二个参数读取文件内容（sub_22340）
    输出：加密后写入新文件（sub_22690），提示 “[+] 加密完成: “，我们可以爆破密钥，而题目提示过我们这是图片png格式，于是我直接尝试png有关的密钥，我在这里写一个全部爆破的密钥

-- coding: utf-8 --

“””
RC4 修改版全密钥爆破脚本
专门针对 4 位数字密钥 (0000-9999)
“””

import sys
import os

def modified_rc4_crypt(key_bytes, data):
“””
修改版 RC4 解密/加密（对称）
特色：密钥流字节 +69 (0x45) 后再异或
“””
# 密钥调度算法 (KSA)
S = list(range(256))
j = 0
key_len = len(key_bytes)

for i in range(256):
    j = (j + S[i] + key_bytes[i % key_len]) & 0xFF
    S[i], S[j] = S[j], S[i]

# 伪随机生成算法 (PRGA) + 修改 (+69)
i = 0
j = 0
out = bytearray()

for byte in data:
    i = (i + 1) & 0xFF
    j = (j + S[i]) & 0xFF
    S[i], S[j] = S[j], S[i]
    k = S[(S[i] + S[j]) & 0xFF]
    # 关键：+69 (0x45)
    keystream_byte = (k + 69) & 0xFF
    out.append(byte ^ keystream_byte)

return bytes(out)

def brute_force_all_keys(cipher_path):
“””
暴力破解 0000-9999，同时检查：

found_count = 0

for num in range(10000):
    key_str = f"{num:04d}"
    key_bytes = key_str.encode()
    
    # 解密
    plain = modified_rc4_crypt(key_bytes, cipher_data)
    
    # 检查 1：是否包含 flag{
    if b'flag{' in plain:
        idx = plain.find(b'flag{')
        flag_candidate = plain[idx:idx+50]  # 提取 flag 前后文
        print(f"\n[!!!] 找到 FLAG！密钥: {key_str}")
        print(f"      内容: {flag_candidate.decode('utf-8', errors='ignore')}")
        
        # 保存
        out_name = f"cracked_{key_str}_flag.txt"
        with open(out_name, 'wb') as f:
            f.write(plain)
        print(f"      已保存完整内容到: {out_name}")
        found_count += 1
        continue  # 继续搜索其他可能的密钥
    
    # 检查 2：是否为常见文件头
    header_checks = [
        (b'\x89PNG', 'png', 'PNG 图片'),
        (b'\xFF\xD8\xFF', 'jpg', 'JPG 图片'),
        (b'GIF8', 'gif', 'GIF 图片'),
        (b'PK\x03\x04', 'zip', 'ZIP 压缩包'),
        (b'7z\xBC\xAF', '7z', '7Z 压缩包'),
        (b'Rar!', 'rar', 'RAR 压缩包'),
        (b'%PDF', 'pdf', 'PDF 文档'),
        (b'\x7fELF', 'elf', 'ELF 可执行文件'),
        (b'MZ', 'exe', 'Windows 可执行文件'),
        (b'flag{', 'txt', '纯文本Flag')
    ]
    
    for magic, ext, desc in header_checks:
        if plain.startswith(magic):
            print(f"\n[+] 密钥 {key_str} -> {desc}")
            
            # 尝试提取可打印字符串看是否有 flag
            try:
                text_part = plain[:500].decode('ascii', errors='ignore')
                if 'flag' in text_part.lower():
                    print(f"      发现 flag 关键字在文本中")
            except:
                pass
            
            out_name = f"recovered_{key_str}.{ext}"
            with open(out_name, 'wb') as f:
                f.write(plain)
            print(f"      已保存为: {out_name}")
            found_count += 1
            break
    
    # 进度显示（每 1000 个显示一次）
    if num % 1000 == 999:
        print(f"    进度: {num+1}/10000 ({(num+1)/100:.0f}%)")

print("-" * 50)
if found_count == 0:
    print("[!] 未找到任何匹配的密钥或文件类型")
    print("[*] 建议：检查原始密文是否正确，或尝试其他偏移量")
else:
    print(f"[+] 共发现 {found_count} 个可疑结果，请检查生成的文件")
