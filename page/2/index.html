title: 入口点包装题复现r-png
date: 2026-2-7

这道题提示图片png格式，推测是根据所给的文件反推代码并最后得出图片
看main函数可知这段反编译代码展示了一个典型的入口点包装器（Entry Point Wrapper）模式，其中出现了三个变量
sub_16A40：这很可能是真正的逻辑入口（flag 检查逻辑通常在此）
sub_24760：检查是否为运行时初始化（Go runtime / C runtime）
qword_59E78：查看交叉引用，可能是关键的全局状态或配置
我先看了24760，经过分析发现这段代码的关键作用是设置程序运行环境，最终调用用户编写的真实 main 函数。，而其中的v32 = (*(int (__fastcall **)(__int64 (__fastcall **)()))(v41 + 40))(p_sub_16A40_1);
p_sub_16A40_1 = 用户编写的 main 函数（即之前提到的 sub_16A40），由此可以知道这关键在于16a40
这是典型的 Rust 编写的文件加密工具，核心逻辑是 RC4 流密码 的变种。
关键识别特征

  1. 算法：修改版 RC4
    密钥调度（KSA）：标准 RC4 初始化（256 次循环，初始化 S-box）
    伪随机生成（PRGA）：标准 RC4 流程，但每个生成的密钥流字节 +69（0x45） 后再与明文异或：
    c
    复制
    buf_1[v13] ^= (keystream_byte + 69);
  2. 密钥约束
    4 位纯数字（如 0123、9999）
    程序会显式检查：”[!] key 必须是 4 位数字，比如 0123\n”
  3. 文件操作流程
    输入：程序从第二个参数读取文件内容（sub_22340）
    输出：加密后写入新文件（sub_22690），提示 “[+] 加密完成: “，我们可以爆破密钥，而题目提示过我们这是图片png格式，于是我直接尝试png有关的密钥，我在这里写一个全部爆破的密钥

“””

import sys

def modified_rc4_crypt(key_bytes, data):
“””
修改版 RC4 解密/加密（对称）
特色：密钥流字节 +69 (0x45) 后再异或
“””
# 密钥调度算法 (KSA)
S = list(range(256))
j = 0
key_len = len(key_bytes)

for i in range(256):
    j = (j + S[i] + key_bytes[i % key_len]) & 0xFF
    S[i], S[j] = S[j], S[i]

# 伪随机生成算法 (PRGA) + 修改 (+69)
i = 0
j = 0
out = bytearray()

for byte in data:
    i = (i + 1) & 0xFF
    j = (j + S[i]) & 0xFF
    S[i], S[j] = S[j], S[i]
    k = S[(S[i] + S[j]) & 0xFF]
    # 关键：+69 (0x45)
    keystream_byte = (k + 69) & 0xFF
    out.append(byte ^ keystream_byte)

return bytes(out)

title: rust逆向解析题目复现rzip
date: 2026-2-7

在main函数里面发现这是一道rust逆向题目，并且main函数里并没有关键信息，里面提到主要代码在compress，接下来查看compress::main
这段代码是一个 LZ77 压缩算法的 Rust 实现。让我为你详细解析其工作原理：
核心算法分析

  1. 命令行参数
    用法: compress （如 compress a.txt a.z）
    需要 3 个参数（程序名 + 输入 + 输出）

  2. 压缩算法（LZ77 变种）
    滑动窗口参数：
    搜索缓冲区: 256 字节（0x100）
    最大匹配长度: 14 字节（从代码中多层 if 判断可见，最大检查到 14）
    最小匹配长度: 3 字节（小于 3 直接输出字面量）
    // 伪代码逻辑
    for i in 0..input.len() {
    // 在 [i-256, i-1] 范围内查找最长匹配
    let (len, dist) = find_longest_match(i);

    if len >= 3 {
    // 编码为 2 字节引用
    output((dist >> 4) | 0x80); // 第一字节：0x80 + 距离高4位
    output((dist << 4) | len); // 第二字节：距离低4位 + 长度
    i += len;
    } else {
    // 直接输出字面量（0x00-0x7F）
    output(input[i]);
    i += 1;
    }

  3. 解压算法（逆向还原）
    如果你需要解压该格式，解压逻辑如下：

    def decompress(data):
    output = bytearray()
    i = 0

    while i < len(data):
    b = data[i]

    if b < 0x80: # 字面量
    output.append(b)
    i += 1
    else: # 反向引用
    if i + 1 >= len(data): break

    b2 = data[i + 1]
    # distance: 高 4 位来自 b1 的低 4 位，低 4 位来自 b2 的高 4 位
    dist = ((b & 0x0F) << 4) | (b2 >> 4)
    # length: b2 的低 4 位（实际值 3-14）

  4. 文件格式特征
    无文件头：纯流式压缩，没有魔数或文件头
    边界: distance 范围 1-256（或 0-255，取决于实现细节），length 范围 3-14
    压缩率显示: 程序会打印 原始大小: X, 压缩后大小: Y, 压缩率: Z%
    根据以上逻辑书写脚本就能对题所给的压缩包解压

#!/usr/bin/env python3
import sys

def decompress(input_file, output_file):
with open(input_file, ‘rb’) as f:
data = f.read()

output = bytearray()
i = 0

while i < len(data):
    first_byte = data[i]
    
    # 判断是否为引用标记（最高位为1，即 0x80-0x8F）
    if first_byte < 0x80:
        # 字面量：直接输出（0x00-0x7F）
        output.append(first_byte)
        i += 1
    else:
        # 引用标记：需要两个字节
        if i + 1 >= len(data):
            print(f"错误：文件截断，在位置 {i} 处缺少第二个字节")
            break
        
        second_byte = data[i + 1]
        
        # 解码距离和长度
        # 第一字节低4位是距离的高4位
        # 第二字节高4位是距离的低4位
        distance = ((first_byte & 0x0F) << 4) | (second_byte >> 4)
        
        # 第二字节低4位是长度（3-14）
        length = second_byte & 0x0F
        
        # 根据代码分析，最小匹配长度应该是3
        # 但如果length小于3，可能是编码问题或者需要特殊处理
        if length < 3:
            # 某些LZ77变体中，length=0,1,2可能有特殊含义，或者直接当作字面量
            # 但根据你的反编译代码，小于3的匹配不会被编码为引用
            print(f"警告：在位置 {i} 发现异常长度 {length}，当作字面量处理")
            output.append(first_byte)
            i += 1
            continue
        
        # 从输出缓冲区向前distance处复制length个字节
        # distance的范围应该是 1-256（滑动窗口大小）
        if distance == 0:
            print(f"错误：在位置 {i} 发现distance为0")
            i += 2
            continue
        
        # 复制匹配的数据
        # 注意：distance=1表示重复最后一个字节（如"AAAA"编码为A + ref(1,3)）
        for _ in range(length):
            if len(output) < distance:
                # 防止缓冲区下溢，虽然正常情况下不该发生
                output.append(0)
            else:
                output.append(output[-distance])
        
        i += 2

with open(output_file, 'wb') as f:
    f.write(output)

print(f"解压完成：{len(data)} 字节 -> {len(output)} 字节")
print(f"压缩率：{(1 - len(data)/len(output))*100:.2f}%" if len(output) > 0 else "N/A")

if name == “main“:
if len(sys.argv) != 3:
print(“用法: python decompress.py <输入文件.z> <输出文件>”)
sys.exit(1)
